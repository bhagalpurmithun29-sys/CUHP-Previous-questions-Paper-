# Data Governance Guide

## 1. Introduction
This guide outlines the operational processes for Data Governance within the CUHP Question Bank. It is designed for the Data Governance Team, Data Stewards, and Executive Users to ensure academic data remains secure, accurate, and compliant.

## 2. Governance Roles & Responsibilities

### 2.1 Super Administrator
* Sets up the infrastructure and core RBAC configurations.
* Manages technical metadata schemas and integration pipelines.
* *Activity*: Monthly review of system audit logs.

### 2.2 Data Governance Team
* Defines data classification policies and retention rules.
* Reviews overall governance compliance via Analytics dashboards.
* *Activity*: Quarterly governance council meetings to update policies.

### 2.3 Data Steward
* Approves changes to Master Data (e.g., adding a new Course).
* Resolves data quality alerts generated by background validation tasks.
* *Activity*: Weekly review of the Data Steward Action Queue.

## 3. Data Lifecycle Management Process

### 3.1 Creation & Ingestion
* All new Question Papers require mandatory metadata tags (Business, Technical, Operational).
* Content is initially marked as `DRAFT` and classified as `INTERNAL`.

### 3.2 Updates & Versioning
* Modifications to metadata or file content trigger a version bump (e.g., v1 -> v2).
* Audit logs capture the `updatedBy` and `updatedAt` properties immutably.

### 3.3 Archival
* Upon reaching the Active Retention limit (e.g., 5 years), an automated cron job sets the status to `ARCHIVED`.
* Archived data is removed from standard search indices but remains accessible in the Analytical Store and via direct queries by authorized roles.

### 3.4 Deletion
* Based on retention policies, data is permanently purged (Hard Delete).
* Cascade deletion is enforced for AI Knowledge Assets linked to the purged Question Paper.

## 4. Metadata Completeness & Quality

* The platform utilizes Background Validation jobs to calculate "Metadata Completeness Scores".
* Papers scoring below 90% are flagged to Data Stewards for remediation.
* **Coverage**: Ensuring all academic courses have at least 3 years of question papers mapped.

## 5. Audit & Compliance
All actions modifying metadata, governance policies, master data, or lifecycle policies generate immutable Audit Logs. These logs are exportable and stored separately for compliance tracking.
